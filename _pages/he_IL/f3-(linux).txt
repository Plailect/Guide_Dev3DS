---
title: "F3 (Linux)"
---

זוהי הרחבה עבור בדיקת כרטיס הSD שלכם עבור שגיאות באמצעות F3.

תלוי בגודל של הSD ובמהירות של המחשב, תהליך זה יכול לקחת מספר שעות!

עמוד זה נועד רק עבור משתמשי לינוקס. אם אתם לא ב-לינוקס, תבדקו את העמודים [H2testw (windows)](h2testw-(windows)) או [F3X (mac)](f3x-(mac)).

#### מה שאתם צריכים

* הגירסה העדכנית של [F3] (https://github.com/AltraMayor/f3/archive/v6.0.zip)

#### הוראות

1. תחלצו את קובץ הזיפ של F3
2. תעשו 'cd' לתיקיית F3
3. תריצו 'make' כדי לקמפל את F3
4. תכניסו את כרטיס ה- SD שלכם אל המחשב
5. תעשו Mount לSD שלכם אם המחשב לא עשה אוטומטית
6. תריצו `./f3write <התיקייה של הSD>`
7. תחכו עד שהתוכנה מסיימת. דוגמה.

		$ ./f3write /media/michel/6135-3363/
		Free space: 29.71 GB
		Creating file 1.h2w ... OK!
		...
		Creating file 30.h2w ... OK!
		Free space: 0.00 Byte
		Average Writing speed: 4.90 MB/s

8. תריצו `./f3read <התיקייה של הSD>`
9. תחכו עד שהתוכנה מסיימת. דוגמה.

		$ ./f3read /media/michel/6135-3363/
		                  SECTORS      ok/corrupted/changed/overwritten
		Validating file 1.h2w ... 2097152/        0/      0/      0
		...
		Validating file 30.h2w ... 1491904/        0/      0/      0

		  Data OK: 29.71 GB (62309312 sectors)
		Data LOST: 0.00 Byte (0 sectors)
			       Corrupted: 0.00 Byte (0 sectors)
			Slightly changed: 0.00 Byte (0 sectors)
			     Overwritten: 0.00 Byte (0 sectors)
		Average Reading speed: 9.42 MB/s


אם הבדיקה מראה את התוצאה ' Data LOST: 0.00 Byte (0 sectors) " הכרטיס שלכם טוב, אתם יכול למחוק את כל קבצי '.h2w' על ה-SD
{: .notice--success}

אם הבדיקה מראה תוצאות אחרות, הכרטיס שלכם עשוי להיות פגום וייתכן שיהיה עליכם להחליף אותו!
{: .notice--danger}
